Name Day Backend API Documentation

The Name Day Backend API is a RESTful service that provides international name-day data by month, date, and country (Poland and Hungary only). It is designed for fast read access, clean routing, and simple deployment.

Useful Links

  • Live frontend demo: https://karoly-git.github.io/name-day-frontend/
  • Backend API: https://name-day-backend-0d74dcea0ed2.herokuapp.com/
  • Swagger / OpenAPI: /api-docs

API Endpoints

Get All Name Days

  • Endpoint: /api/v1/namedays
  • Method: GET
  • Description: Retrieve all name-day data.
  • Example:
    /api/v1/namedays

Get Name Days by Month

  • Endpoint: /api/v1/namedays/:month
  • Method: GET
  • Parameter: month â€” lowercase month name
  • Example:
    /api/v1/namedays/january

Get Name Days by Month and Date

  • Endpoint: /api/v1/namedays/:month/:date
  • Method: GET
  • Parameters: month, date
  • Example:
    /api/v1/namedays/january/1

Get Name Days by Month, Date, and Country

  • Endpoint: /api/v1/namedays/:month/:date/:country
  • Method: GET
  • Supported countries: pl, hu
  • Example:
    /api/v1/namedays/january/1/pl

Health Check

  • Endpoint: /health
  • Method: GET
{   "status": "ok" }

Error Handling

Invalid or unavailable data returns a 404 Not Found response.

{   "error": "Data not found" }
