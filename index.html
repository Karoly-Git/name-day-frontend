Name Day API Documentation

Welcome to the Name Day API documentation. This API provides name day data for various months of Hungary and Poland. Name days are special days in some cultures where individuals celebrate their given names based on a calendar of saints or historical figures.

1. Base URL

https://nameday-b9bd663a9ed7.herokuapp.com/

2. Endpoints

Get All Name Days

  • Endpoint: /
  • Method: GET
  • Description: Retrieve name day data for all months and countries.
  • Example Request:
    https://nameday-b9bd663a9ed7.herokuapp.com/
  • Example Response:
    {
      "january": {
        "1": { "hu": ["Fruzsina"], "pl": ["Mieczysław", "Mieczysława", "Mieszko"] },
        ...
      }
    }

Get Name Days by Month

  • Endpoint: /:month
  • Method: GET
  • Description: Retrieve name day data for a specific month.
  • Parameters: month (string): The name of the month in lowercase (e.g., "january").
  • Example Request:
    https://nameday-b9bd663a9ed7.herokuapp.com/january
  • Example Response:
    {
      "1": { "hu": ["Fruzsina"], "pl": ["Mieczysław", "Mieczysława", "Mieszko"] },
      ...
    }

Get Name Days by Month and Date

  • Endpoint: /:month/:date
  • Method: GET
  • Description: Retrieve name day data for a specific day within a month.
  • Parameters: month (string): The name of the month in lowercase (e.g., "january"). date (string): The day of the month as a number (e.g., "1").
  • Example Request:
    https://nameday-b9bd663a9ed7.herokuapp.com/january/1
  • Example Response:
    {
      "hu": ["Fruzsina"],
      "pl": ["Mieczysław", "Mieczysława", "Mieszko"]
    }

Get Name Days by Month, Date, and Country

  • Endpoint: /:month/:date/:country
  • Method: GET
  • Description: Retrieve name day data for a specific day within a month and a specific country.
  • Parameters: month (string), date (string), country (string).
  • Example Request:
    https://nameday-b9bd663a9ed7.herokuapp.com/january/1/hu
  • Example Response:
    ["Fruzsina"]

3. Response Format

All responses are in JSON format. Successful responses return data corresponding to the requested parameters.

{
  "error": "Data not found"
}

Error Handling

The API will return appropriate HTTP status codes and error messages for invalid requests.

  • 400 Bad Request: Invalid request format or parameters.
  • 404 Not Found: Data not found for the requested parameters.
  • 500 Internal Server Error: Unexpected server errors.

Rate Limiting

This API may have rate limiting in place to prevent abuse.
